# 项目设置指南

## 第一步：安装依赖

在项目根目录运行：

```bash
npm install
```

## 第二步：配置API密钥

1. 创建 `.env` 文件（参考 `.env.example`）

2. 申请并配置API密钥：

   **百度地图API密钥**：
   - 访问 https://lbsyun.baidu.com/
   - 注册/登录后创建应用
   - 获取AK（Access Key）
   - 填入 `.env` 文件的 `VITE_MAP_API_KEY`

   **和风天气API密钥**：
   - 访问 https://dev.qweather.com/
   - 注册/登录后创建应用
   - 获取Key（只需要Key，不需要Secret）
   - 填入 `.env` 文件的 `VITE_WEATHER_API_KEY`
   - **注意**：Secret用于签名验证，免费版API不需要Secret

3. 在API服务商配置域名白名单（生产环境需要）

## 第三步：启动开发服务器

```bash
npm run dev
```

浏览器访问：http://localhost:5173

## 第四步：使用应用

1. 在输入框中输入经过地点（如：北京、上海）
2. 选择每个地点的到达日期和时间
3. 点击"添加经过点"可以添加多个地点
4. 点击"查询天气"按钮
5. 查看每个地点的天气信息

## 常见问题

### 1. API调用失败

- 检查 `.env` 文件中的API密钥是否正确
- 检查API服务商的调用限制
- 检查网络连接

### 2. CORS跨域问题

- 开发环境：Vite已配置代理，应该不会有问题
- 生产环境：需要在API服务商配置域名白名单

### 3. 天气数据不显示

- 检查和风天气API密钥是否正确
- 检查日期是否在未来7天内（和风天气免费版支持7天预报）
- 查看浏览器控制台的错误信息

## 构建生产版本

```bash
npm run build
```

构建产物在 `dist` 目录，可以部署到任何静态托管服务。
